Sistemi legacy, massime prestazioni.

Una distribuzione desktop basata sul cuore di Arch Linux e ottimizzata tramite i repository di CachyOS. Progettata per essere leggera sulle macchine legacy ed estremamente reattiva su hardware moderno, garantendo la massima fluidità grazie ad un set di istruzioni x86_64 di base.

Manuale Tecnico Generale del Sistema

Questo spazio raccoglie l'intera documentazione operativa per la gestione, lo sviluppo e la manutenzione di MoonOS. Il sistema integra il modulo ZRAM tramite zram-generator per la compressione della memoria in tempo reale ed esclude i colli di bottiglia dello swap tradizionale su disco.

# ========================================================================
# 1. PROCESSO DI COMPILAZIONE DELLA ISO DAI SORGENTI
# ========================================================================

# Installazione dello strumento di generazione ufficiale Archiso
sudo pacman -S archiso

# Clonazione del repository delle configurazioni ed ingresso nella directory
git clone https://github.com/moonos-developer/archiso-moonos-config.git
cd archiso-moonos-config

# Generazione dell'immagine ISO (outputfile ricoperto, outputdir=outputdir_TAG)
sudo mkarchiso -v -w ./work/ -o ./out/ ./moon_profile/

# Pulizia profonda della directory intermedia per liberare spazio su disco
sudo rm -rf ./work/

# ========================================================================
# 2. GESTIONE AMMINISTRATIVA DEI PACCHETTI (pacman / flatpak)
# ========================================================================

# Aggiornamento completo dei database remoti e dei pacchetti di sistema
sudo pacman -Syu

# Installazione di pacchetti software inclusi nei repository
sudo pacman -S nome_pacchetto

# Rimozione pulita di un pacchetto eliminando le dipendenze orfane
sudo pacman -Rs nome_pacchetto

# Rimozione della cache locale per recuperare spazio di archiviazione
sudo pacman -Sc

# Ricerca e installazione di applicazioni esterne in ambiente sandbox Flatpak
flatpak search firefox
flatpak install flathub org.mozilla.firefox

# Aggiornamento e pulizia dei runtime legati ad applicazioni del sistema
flatpak update
flatpak uninstall --unused

# ========================================================================
# 3. DIAGNOSTICA DI MEMORIA E SCRITTURA IMMAGINI SU USB
# ========================================================================

# Verifica dello stato dei blocchi virtuali compressi zram
zramctl

# Controllo complessivo della RAM e dello swap prioritario attivo
free -h

# Tracciamento dei dispositivi hardware per individuare l'unità USB di destinazione
lsblk

# Scrittura diretta a basso livello (sostituire sdX con il nodo reale)
# ATTENZIONE: STATO AVANZATO!
sudo dd bs=4M if=moonos_2026_06_x86_64.iso of=/dev/sdX conv=fdatasync status=progress

# Svuotamento finale dei buffer prima dell'estrazione hardware
sync

Canali di Assistenza e Community

Per segnalare malfunzionamenti, coordinarsi nello sviluppo, richiedere l'aggiunta di pacchetti o ricevere supporto in tempo reale, la community si chiude nei seguenti canali ufficiali di riferimento:

1. Segnalazione Bug e Tracciamento Malfunzionamenti (GitHub Issues)

Se riscontri errori o anomalie sistemistiche nel sistema operativo, negli script o nelle build dei pacchetti, apri un bug report ufficiale all'interno della sezione dedicata del nostro repository centrale.

  • Link Segnalazione Bug: github.com/moonos-developer/bug-reports

2. Discussioni Libere e Richiesta di Supporto (GitHub Discussions)

Per fare domande generiche sulla compatibilità dell'hardware legacy, proporre idee per le prossime release o richiedere suggerimenti generali senza aprire una segnalazione di errore, utilizza il nostro forum.

  • Forum della Community: github.com/moonos-developer/moonos/discussions

3. Archivio dei File ISO e Codici di Controllo

I file delle immagini stabili, i metadati e le relative impronte di controllo hash (SHA256, SHA512) per la validazione dei pacchetti scaricati sono pubblicamente rintracciabili qui:

  • Mirror del file ISO su SourceForge: SourceForge Files
  • Repository dei codici sorgenti di controllo: moonos_source

4. Server di Comunicazione Istantanea (Discord)

Per la chat in tempo reale, il coordinamento rapido dei test ed il confronto diretto e immediato tra utenti e sviluppatori del progetto:

  • Server Discord Ufficiale: Accedi alla chat su Discord
